# App scaffold — top-level directory justification

The `feat/app-scaffold` branch adds the Gay Cruise Bingo web app, which introduces a top-level directory beyond the canonical repo set. Per the [Code Modification Rules](../docs/agents/code-modification-rules.md) requirement — "Do not introduce new top-level directories without documented justification in `AGENTS.md` or a `plans/` entry" — its rationale is recorded here rather than in `AGENTS.md`. It is also whitelisted in `.repo-template.yml` (`extra_top_level_dirs: [public]`), but the CI whitelist only stops the lint gate from failing; it is not the documented justification the rule asks for.

- **`public/`** — Vite static assets served verbatim by Firebase Hosting: the favicon, the PWA manifest icons (`pwa-192.png`, `pwa-512.png`, `apple-touch-icon.png`), the fallback Open Graph image (`og-default.png`), and the service worker (`sw.js`). Vite copies this directory into the build output untouched, so it cannot live under `src/`.

**`cloud-run/`** — no longer present. It held the Phase 1 Open Graph image renderer (`cloud-run/og-renderer`), a standalone Express + Playwright service deployed to Cloud Run that rendered themed retina share images on demand for the `share` Cloud Function's link unfurls. ADR 0005 marked it as not used at launch (Share Cards are generated on-device instead), and #39 deleted the directory, the `share` Function, and the `/s/**` hosting rewrite together — it is dropped from `extra_top_level_dirs` accordingly.

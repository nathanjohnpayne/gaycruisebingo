# App scaffold — top-level directory justification

The `feat/app-scaffold` branch adds the Gay Cruise Bingo web app, which introduces two top-level directories beyond the canonical repo set. Per the [Code Modification Rules](../docs/agents/code-modification-rules.md) requirement — "Do not introduce new top-level directories without documented justification in `AGENTS.md` or a `plans/` entry" — their rationale is recorded here rather than in `AGENTS.md`. Both are also whitelisted in `.repo-template.yml` (`extra_top_level_dirs: [public, cloud-run]`), but the CI whitelist only stops the lint gate from failing; it is not the documented justification the rule asks for.

- **`public/`** — Vite static assets served verbatim by Firebase Hosting: the favicon, the PWA manifest icons (`pwa-192.png`, `pwa-512.png`, `apple-touch-icon.png`), the fallback Open Graph image (`og-default.png`), and the service worker (`sw.js`). Vite copies this directory into the build output untouched, so it cannot live under `src/`.
- **`cloud-run/`** — the Phase 1 Open Graph image renderer (`cloud-run/og-renderer`), a standalone Express + Playwright service deployed to Cloud Run. It renders themed retina share images on demand, and the `share` Cloud Function points link unfurls at it. It is a separate top-level package rather than part of `functions/` because its Playwright/Chromium footprint and Cloud Run runtime differ from the Cloud Functions bundle.

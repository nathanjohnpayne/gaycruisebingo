# Prompt pool reseed + stratified Board composition (`seed-and-composition`)

The Prompt pool is replaced with a canonical 87-entry set, and Board dealing changes from uniform-random sampling to a stratified spicy/tame sample, so a dealt card mixes both categories instead of possibly landing all one kind of joke. "Spicy" is the ðŸ”ž-tagged category, stored as a `spicy: boolean` on every Prompt â€” the emoji is a category tag applied at authoring/seeding time, never present in the Prompt's own display text. This spec supersedes `w1-event-seed`'s narrower ~30â€“50 "dense pool" band (that spec's other claims â€” the ADR 0001 Claim Mode naming, the ADR 0004 `blackoutEnabled` delete sentinel, the `ADMIN_UID` roster flow â€” are unchanged and still covered there).

## Contract

- `src/data/seed.ts` â€” `SEED_ITEMS: { text: string; spicy: boolean }[]` holds the canonical 87-entry pool (24 spicy, 63 tame), replacing the old 32-entry `string[]`. `FREE_TEXT` is `'Complain about circuit music'` (lowercased "circuit music", was "Circuit Music").
- `src/types.ts` â€” `ItemDoc.spicy: boolean` records the category on every Prompt document. `EventDoc['settings'].spicyRatio?: number` is the optional per-Event override of the target spicy share; it is read defensively (`typeof === 'number'`) at the one call site that needs it, the same way `settings.reportHideThreshold` already is, even though both are typed on `EventDoc`. Absent on an Event doc seeded before this field existed, in which case `dealBoard`'s own default applies.
- `src/game/logic.ts` â€” `DealItem` carries `spicy: boolean`. `dealBoard(pool, freeText, seed, spicyRatio = 0.4)` deals the frozen 5Ã—5 Board via a stratified sample: it shuffles the spicy and tame subsets of `pool` independently (both with the same seeded PRNG instance, so the whole deal stays deterministic per seed), takes `round(24 * spicyRatio)` from spicy and the remainder from tame, backfills from whichever category has more remaining when the other runs short, and re-shuffles the chosen 24 across grid positions so spicy and tame interleave rather than cluster together. The pre-existing `pool.length < MIN_POOL` (24) guard fires before any stratification and is unaffected by the ratio or the backfill.
- `src/data/api.ts` â€” `joinAndDeal` maps the fetched pool to `DealItem[]` including `spicy`, reads `settings.spicyRatio` off the same already-fetched event doc as `reportHideThreshold`, and passes it (or the 0.4 default when absent) as `dealBoard`'s 4th argument. `addItem(uid, text, spicy = false)` writes the `spicy` flag on a user-submitted Prompt; a Prompt added without checking the ðŸ”ž toggle defaults to tame.
- `src/components/ItemPool.tsx` â€” a ðŸ”ž checkbox next to the add-Prompt input lets a Player mark their submission as spicy; it resets to unchecked after a successful add, alongside the text field.
- `scripts/seed.mjs` â€” `EVENT_SEED.settings` carries both `reportHideThreshold: 4` and `spicyRatio: 0.4`. `ITEMS` holds the same 87-entry `{ text, spicy }` pool as `SEED_ITEMS`. The `seed()` function now replaces rather than appends: it deletes every existing doc in `events/{id}/items` before writing the 87 new ones, so re-running the script never leaves stale Prompts from a prior pool mixed in with the current one. Doc ids stay content-hashed on `text` alone (unchanged from before), and each written doc now also carries `spicy`.

## Acceptance criteria

- **Given** the seed script runs on a fresh Event, **when** it completes, **then** `events/{id}/items` holds exactly the 87 active Prompts with correct `spicy` flags, the Free Space is not among them, and re-running the script replaces rather than duplicates the pool. (Tests: "exports the same 87-entry { text, spicy } pool as src/data/seed.ts SEED_ITEMS"; the delete-then-write replace semantics itself needs a live/emulated Firestore and is out of scope for this unit layer â€” see Test coverage below.)
- **Given** the full pool, **when** a Board is dealt, **then** it has 24 non-free Squares (10 spicy / 14 tame at the default 0.4 ratio) plus the marked Free Space center, deterministic per seed, with the spicy Squares interleaved (not clustered) across the grid. (Tests: "deals 25 cells: a marked free center + 24 non-free"; "deals exactly 10 spicy / 14 tame among the 24 non-free cells at the default ratio"; "is deterministic per seed and varies across seeds"; "interleaves spicy cells across at least 3 different grid rows (not clustered)".)
- **Given** a category is short, **then** backfill keeps the deal at 24 total; **given** the active non-free pool is below `MIN_POOL` (24), **then** `dealBoard` throws exactly like the pre-existing guard. (Tests: "backfills tame when spicy is short: 5 spicy + 40 tame -> 5 spicy + 19 tame, no throw"; "throws when the active pool has fewer than MIN_POOL (24) prompts, same as the pre-existing guard".)

## Test coverage

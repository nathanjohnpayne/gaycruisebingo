# Iconography: lucide-react for chrome/controls (d15-icons-lucide)

Feature: adopts `lucide-react` for the app's chrome and controls per `plans/daily-cards-spec.md` § "Iconography — Lucide". The rule, verbatim: **Lucide for chrome and controls, emoji for camp.** Navigation, buttons, and system affordances (tab bar, More menu rows, the claim sheet's segments/affordances, the locked-Day badge) switch to `lucide-react` glyphs; theme emoji (`ThemeMeta.emoji`), party names, Moments, toast lead-ins, and Feed source badges stay emoji unchanged — that is the app's personality and out of scope here.

## Contract

- `package.json` — `lucide-react` added as a dependency. Every icon is imported by name (`import { Grid3x3 } from 'lucide-react'`), never the whole library, so bundling stays tree-shakeable.
- `src/components/TabBar.tsx` — Card renders `Grid3x3`, Feed renders `Radio`, Ranks renders `Trophy` (className `tab-icon`), each ahead of its plain-text label. The More tab is unchanged in kind (the Player's avatar `<img>` when `morePhotoURL` is set) except its signed-out fallback glyph is now the `lucide-react` `Ellipsis` icon (className `tab-ellipsis`) instead of a literal `⋯` character. `src/components/tabs.ts`'s frozen `TabId`/`TABS`/`visibleTabs()` contract is untouched.
- `src/components/More.tsx` — `MoreRow` (the shared row shape for Cruise schedule / Suggest a square / How to play / Admin) gains a required `icon` prop rendered at the row's leading edge (className `more-row-icon`) and an unconditional trailing `ChevronRight` (className `more-row-chevron`), since every `MoreRow` opens a sub-panel. Icons: schedule `CalendarDays`, suggest `Lightbulb`, how to play `GraduationCap`, admin `Wrench`. The Theme section header gains a leading `Palette` icon. The Install row (a plain `.more-row` button outside `MoreRow`, since it fires an install prompt rather than opening a sub-panel) gains a leading `Download` icon; the Sign out row (also outside `MoreRow`, `.more-row-quiet`) gains a leading `LogOut` icon. Neither Install nor Sign out gets a chevron — neither navigates to a sub-panel. Text size (`#215`) has no row yet (the Theme/Play section still just reserves its slot per `specs/d15-more-menu.md` § Out of scope), so the spec's `a-large-small` mapping has nothing to attach to until that ticket lands; the 18+ row (`AcceptableUse`, `shield-alert`) is untouched — it is not in this ticket's file list and already carries its own hand-inlined guidelines glyph, unrelated to this pass.
- `src/components/BugReport.tsx` — `BugIcon` (used by both the `'floating'` and `'row'` variants, `className="bug-report-icon"` preserved across both) now renders `lucide-react`'s `Bug` glyph instead of a hand-inlined `<svg>` path. `Nav.tsx`'s sign-out button, called out in the issue as a hand-inlined `log-out` glyph, has since relocated into `More.tsx`'s plain-text Sign out row (`specs/d15-tab-contract.md` / `specs/d15-more-menu.md`) — there is no hand-inlined SVG left in `Nav.tsx` to migrate; this ticket's `LogOut` icon on that row (above) is the current equivalent.
- `src/components/ProofSheet.tsx` (the claim sheet) — the proof-type segmented control's three buttons render a leading Lucide glyph (className `seg-btn-icon`) ahead of their text label: photo `Camera`, audio `Mic`, text `PenLine`; their emoji prefixes (`📷`/`🎙`/`✍️`) are dropped. The photo-capture affordances swap their emoji prefixes for `Camera` (Take photo) and `Images` (Library), className `photo-affordance-icon`. The sheet gains a `.sheet-header` wrapper (title + an icon-only `X` dismiss button, className `sheet-dismiss`, `aria-label="Close"`) alongside the existing "Cancel" action button — both call the same `onClose`. This wrapper is opt-in per sheet; other sheets (`BugReport`, `MorePanel`, `AcceptableUse`) keep a bare `.sheet-title` and are unchanged.
- `src/components/Board.tsx` — the locked-Day preview's unlock badge (`.day-lock-badge`) renders a Lucide `Lock` icon (className `day-lock-icon`) instead of the `🔒` emoji. Audio playback in the Feed (`ProofFeed.tsx`'s `<audio controls>`) uses the browser's native transport controls, not a custom in-app play affordance — there is no hand-rolled "play" button in the codebase for the spec's optional `play` glyph to replace, so this ticket makes no change there (a sensible default per the issue's "Decisions are RESOLVED" instruction, not a new decision point).
- `src/index.css` — new sizing rules for every className introduced above (`tab-icon`/`tab-ellipsis`, `more-row-icon`/`more-row-chevron`/`more-section-icon`, `sheet-header`/`sheet-dismiss`, `seg-btn-icon`, `photo-affordance-icon`, `day-lock-icon`), each scoped narrowly so no existing selector's box model changes.

## Acceptance criteria

- Given the tab bar, when it renders, then Card/Feed/Ranks show the `Grid3x3`/`Radio`/`Trophy` Lucide icons and the signed-out More fallback renders the `Ellipsis` Lucide icon (not a literal `⋯` character).
- Given the More menu, when it renders, then the Theme header, Cruise schedule, Suggest a square, How to play, Install the app, the admin-only Admin row, and Sign out each show their spec-mapped Lucide icon, and every sub-panel-opening row (schedule/suggest/how-to-play/admin) shows a trailing chevron.
- Given the claim sheet, when it renders, then its three segments show `Camera`/`Mic`/`PenLine`, the photo affordances show `Camera`/`Images`, and an icon-only `X` dismiss control closes the sheet exactly like the existing Cancel button.
- Given the locked-Day preview, when it renders, then its unlock badge shows a Lucide `Lock` icon, not the `🔒` emoji.
- Given a Theme, Moment, toast, or Feed source badge, when it renders, then it is still emoji, unchanged by this ticket.

  - `More`: with the same stubbed-children pattern `d15-more-menu.test.tsx` established, asserts each row's icon class is present (`.more-row-icon` on schedule/suggest/how-to-play/admin/install/sign-out) and that schedule/suggest/how-to-play/admin additionally render `.more-row-chevron`, while install/sign-out do not.

  - A grep assertion over `src/components/*.tsx` confirming no hand-inlined `<svg>` duplicates a `lucide-react` icon this ticket introduces (i.e., `BugReport.tsx` no longer defines its own bug path data).
- `src/components/Board.test.tsx` (extended) — the existing "Locked-Day preview" describe block gains one assertion that `.day-lock-badge` contains a `.day-lock-icon` `<svg>` and no `🔒` text node.

## Out of scope (later tickets)

Text size's `a-large-small` row (`#215`, not yet built) and the 18+ row's `shield-alert` (`AcceptableUse.tsx`, not in this ticket's file list) are both spec-mapped icons with no current attachment point or owning ticket in this pass — see the Contract section above for why each is a sensible default rather than an invented row/file. Device/browser-chrome icons (`signal`, `battery-full`, `chevron-left`, `share`, `copy`) are wireframes-only per spec and correspond to no real app UI.
